# ⚙️ STEP 1 — Install GraphQL in NestJS

```bash
npm install @nestjs/graphql @nestjs/apollo graphql apollo-server-express
```

---

# ⚙️ STEP 2 — Configure GraphQL Module

In `app.module.ts`:

```ts
import { GraphQLModule } from '@nestjs/graphql';
import { ApolloDriver, ApolloDriverConfig } from '@nestjs/apollo';

GraphQLModule.forRoot<ApolloDriverConfig>({
  driver: ApolloDriver,
  autoSchemaFile: true,
});
```

---

# 🧠 STEP 3 — Think in GraphQL Terms (VERY IMPORTANT)

GraphQL = **types + resolvers**

👉 Not controllers anymore
👉 You build:

* Object Types (like DTOs)
* Resolvers (like controllers)

---

# 🏗️ STEP 4 — Create Your GraphQL Types

## 🔥 Match Type

```ts
import { ObjectType, Field, Int } from '@nestjs/graphql';

@ObjectType()
export class MatchType {
  @Field(() => Int)
  id: number;

  @Field()
  homeTeam: string;

  @Field()
  awayTeam: string;

  @Field(() => Int, { nullable: true })
  homeScore?: number;

  @Field(() => Int, { nullable: true })
  awayScore?: number;
}
```

---

## 🔥 Prediction Type

```ts
@ObjectType()
export class PredictionType {
  @Field(() => Int)
  id: number;

  @Field(() => Int)
  homeScore: number;

  @Field(() => Int)
  awayScore: number;

  @Field(() => Int)
  points: number;

  @Field(() => MatchType)
  match: MatchType;
}
```

---

## 🔥 User Type

```ts
@ObjectType()
export class UserType {
  @Field(() => Int)
  id: number;

  @Field()
  name: string;

  @Field(() => Int)
  points: number;

  @Field(() => [PredictionType])
  predictions: PredictionType[];
}
```

# 🧠 STEP 5 — Build Your FIRST Resolver

👉 Think of resolver like:

> “GraphQL Controller”

---

## 🔥 1. Matches + Predictions Resolver

```ts
import { Resolver, Query } from '@nestjs/graphql';

@Resolver()
export class MatchResolver {
  constructor(private prisma: PrismaService) {}

  @Query(() => [MatchType])
  async matches() {
    return this.prisma.match.findMany({
      include: {
        predictions: {
          include: {
            user: true,
          },
        },
      },
    });
  }
}
```

---

# 🧠 STEP 6 — Leaderboard Resolver

```ts
@Resolver()
export class LeaderboardResolver {
  constructor(private prisma: PrismaService) {}

  @Query(() => [UserType])
  async leaderboard() {
    return this.prisma.user.findMany({
      orderBy: {
        points: 'desc',
      },
      take: 10,
      include: {
        predictions: true,
      },
    });
  }
}
```

---

# 🧠 STEP 7 — User Dashboard Resolver (IMPORTANT)

👉 This is where GraphQL shines 🔥

```ts
@Resolver()
export class UserResolver {
  constructor(private prisma: PrismaService) {}

  @Query(() => UserType)
  async userDashboard(@Args('userId', { type: () => Int }) userId: number) {
    return this.prisma.user.findUnique({
      where: { id: userId },
      include: {
        predictions: {
          include: {
            match: true,
          },
        },
      },
    });
  }
}
```

# ⚠️ IMPORTANT BEST PRACTICES

## 1. DO NOT overload resolvers

Bad:

```ts
huge nested logic ❌
```

Good:

```ts
Call service layer ✅
```

---

## 2. Reuse your services

```ts
constructor(private predictionService: PredictionService)
```

---

## 3. Keep REST for:

* Auth
* Writes (POST, PUT)

Use GraphQL for:

* Reads
